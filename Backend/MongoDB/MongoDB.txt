Find by age
db.students.find({ age: 22 })

11. Comparison Operators-------------------------------------- 
Greater than
db.students.find({ age: { $gt: 21 } })
Meaning

Find students whose age is greater than 21.

Less than
db.students.find({ age: { $lt: 23 } })

Meaning
Find students whose age is less than 23.

Greater than equal
db.students.find({ age: { $gte: 22 } })

Meaning
Find students whose age is greater than or equal to 22.

Less than equal
db.students.find({ age: { $lte: 22 } })

Meaning
Find students whose age is less than or equal to 22.

Aggregation in MongoDB -----------------------------------------------------------------------------------------------

What is Aggregation?

Aggregation is used to:

process data
filter data
group data
calculate values
transform data

--------

Aggregation Syntax

db.collection.aggregate([
   { stage1 },
   { stage2 },
   { stage3 }
])

--------

IMPORTANT CONCEPT — Pipeline

db.students.aggregate([
  { $match: { course: "MERN" } },
  { $project: { name: 1, age: 1 } }
])

Pipeline Flow

STEP 1:
Filter MERN students

STEP 2:
Show only name and age

---------

Aggregation Stages

1. $match

Purpose
Filters documents.

Same as find().

Example
db.students.aggregate([
  {
    $match: {
      course: "MERN"
    }
  }
])
Output

Returns only MERN students.

---------

2. $project

Purpose
Show selected fields.

Example
db.students.aggregate([
  {
    $project: {
      name: 1,
      marks: 1
    }
  }
])

Output

{
  "name": "Rohit",
  "marks": 85
}

---------

3. $sort

Purpose
Sort data.

Ascending
db.students.aggregate([
  {
    $sort: {
      marks: 1
    }
  }
])
Descending
db.students.aggregate([
  {
    $sort: {
      marks: -1
    }
  }
])

Explanation

 1 = ascending
-1 = descending

---------

4. $limit

Purpose
Limit records.

Example
db.students.aggregate([
  {
    $limit: 2
  }
])
Output

Returns only 2 documents.

--------- 

5. $skip

Purpose
Skip documents.

Example
db.students.aggregate([
  {
    $skip: 2
  }
])
Output

Returns only 2 documents.

---------

6. $group (VERY IMPORTANT)
   This is the HEART of aggregation.

Purpose
Group documents together.

Example 1 — Group by Course
db.students.aggregate([
  {
    $group: {
      _id: "$course"
    }
  }
])
Output
{ "_id": "MERN" }
{ "_id": "Java" }
{ "_id": "Python" }

db.students.aggregate([
  {
    $group: {
      _id: "$course",
      totalStudents : { $sum : 1 }
    }
  }
]);
Output
{ "_id": "MERN", "totalStudents": 2 }
{ "_id": "Java", "totalStudents": 2 }
{ "_id": "Python", "totalStudents": 1 }

Aggregation Operators with $group

---------

7. $sum
Purpose

Total calculation.

Example — Total Fees
db.students.aggregate([
  {
    $group: {
      _id: null,
      totalFees: {
        $sum: "$fees"
      }
    }
  }
])
output

{ "_id": null, "totalFees": 232000 }

Group-wise Total
db.students.aggregate([
  {
    $group: {
      _id: "$course",
      totalFees: {
        $sum: "$fees"
      }
    }
  }
])
output

{ "_id": "Java", "totalFees": 82000 }
{ "_id": "MERN", "totalFees": 105000 }
{ "_id": "Python", "totalFees": 45000 }

1. Multi-Stage Pipeline Thinking

Example
db.orders.aggregate([
  {
    $match: {
      status: "completed"
    }
  },
  {
    $group: {
      _id: "$category",
      totalSales: {
        $sum: "$amount"
      }
    }
  },
  {
    $sort: {
      totalSales: -1
    }
  }
])

VERY IMPORTANT CONCEPT

Aggregation ALWAYS works step by step.

Each stage takes previous stage output.
 
output :-
{
  _id: 'Electronics',
  totalSales: 12000
}
{
  _id: 'Fashion',
  totalSales: 4500
}

---------

2. $cond (Condition / IF ELSE)
Purpose

Conditional logic inside aggregation.

Example
db.orders.aggregate([
  {
    $project: {
      customer: 1,
      amount: 1,
      orderType: {
        $cond: {
          if: { $gte: ["$amount", 5000] },
          then: "Premium",
          else: "Normal"
        }
      }
    }
  }
])

Output
{
  "customer": "Rohit",
  "orderType": "Premium"
}

---------

3. $sum with Condition
Example

Count completed orders only.

db.orders.aggregate([
  {
    $group: {
      _id: null,
      completedOrders: {
        $sum: {
          $cond: [
            { $eq: ["$status", "completed"] },
            1,
            0
          ]
        }
      }
    }
  }
])

output
{
  "_id": null,
  "completedOrders": 3
}

---------

4. $multiply
Purpose

Multiply values.

Example
db.orders.aggregate([
  {
    $project: {
      customer: 1,
      totalPrice: {
        $multiply: ["$amount", "$quantity"]
      }
    }
  }
])

you can do $divide, $subtract too

---------

5. $add
Example
db.orders.aggregate([
  {
    $project: {
      customer: 1,
      finalAmount: {
        $add: ["$amount", 500]
      }
    }
  }
])


---------

6. $facet (VERY IMPORTANT ADVANCED)

Purpose
Run multiple pipelines together.

Example

db.orders.aggregate([
  {
    $facet: {
      completedOrders: [
        { $match: { status: "completed" } }
      ],
      pendingOrders: [
        { $match: { status: "pending" } }
      ]
    }
  }
])

7. $size
Purpose

Count array items.

Dataset
db.users.insertOne({
  name: "Rohit",
  skills: ["React", "Node", "MongoDB"]
})

Example
db.users.aggregate([
  {
    $project: {
      name: 1,
      totalSkills: {
        $size: "$skills"
      }
    }
  }
])
