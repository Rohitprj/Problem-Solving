🧠 action kaha se aata hai in fulfilled?

Jab tum Redux Toolkit ka createAsyncThunk use karte ho:

export const fetchData = createAsyncThunk(
  'data/fetch',
  async () => {
    const res = await fetch('/api/data')
    return res.json()
  }
)

👉 Jo tum return karte ho async function ke andar
👉 wahi automatically action.payload ban jaata hai

🔥 Flow samajh lo:
Tum dispatch karte ho:
dispatch(fetchData())
Redux internally 3 actions fire karta hai:
fetchData.pending
fetchData.fulfilled
fetchData.rejected
Jab API success hoti hai:
return res.json()

👉 Ye value → action.payload ban jaati hai

✅ Isliye yahan:
.addCase(fetchData.fulfilled, (state, action) => {
  state.data = action.payload
})

👉 action.payload = API ka response

🚀 Real Use Case: Login + Register Slice

Chalo ab ek practical example banate hain 🔥

📁 authSlice.js
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit'

// 🔹 Register API
export const registerUser = createAsyncThunk(
  'auth/register',
  async (userData, thunkAPI) => {
    try {
      const res = await fetch('/api/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(userData),
      })

      const data = await res.json()

      if (!res.ok) {
        return thunkAPI.rejectWithValue(data.message)
      }

      return data // 👉 goes to action.payload
    } catch (error) {
      return thunkAPI.rejectWithValue(error.message)
    }
  }
)

// 🔹 Login API
export const loginUser = createAsyncThunk(
  'auth/login',
  async (userData, thunkAPI) => {
    try {
      const res = await fetch('/api/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(userData),
      })

      const data = await res.json()

      if (!res.ok) {
        return thunkAPI.rejectWithValue(data.message)
      }

      return data // 👉 payload
    } catch (error) {
      return thunkAPI.rejectWithValue(error.message)
    }
  }
)

// 🔹 Initial State
const initialState = {
  user: null,
  loading: false,
  error: null,
}

// 🔹 Slice
const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    logout: (state) => {
      state.user = null
    },
  },

  extraReducers: (builder) => {
    builder

      // ✅ Register
      .addCase(registerUser.pending, (state) => {
        state.loading = true
        state.error = null
      })
      .addCase(registerUser.fulfilled, (state, action) => {
        state.loading = false
        state.user = action.payload   // 👈 API response
      })
      .addCase(registerUser.rejected, (state, action) => {
        state.loading = false
        state.error = action.payload  // 👈 error message
      })

      // ✅ Login
      .addCase(loginUser.pending, (state) => {
        state.loading = true
        state.error = null
      })
      .addCase(loginUser.fulfilled, (state, action) => {
        state.loading = false
        state.user = action.payload   // 👈 user data / token
      })
      .addCase(loginUser.rejected, (state, action) => {
        state.loading = false
        state.error = action.payload
      })
  },
})

export const { logout } = authSlice.actions
export default authSlice.reducer
🎯 Real Life Flow (Login Example)
🧑 User login form submit karta hai:
dispatch(loginUser({ email, password }))
⚙️ Backend se response aata hai:
{
  "user": { "id": 1, "name": "Rahul" },
  "token": "abcd123"
}
🔥 Redux flow:
pending → loader ON
API success → fulfilled
return data → action.payload
🧠 Final state:
{
  user: { id: 1, name: "Rahul" },
  loading: false,
  error: null
}
⚡ Key Takeaways
action automatically aata hai Redux se
action.payload = jo tum thunk se return karte ho
rejectWithValue() = custom error payload bhejne ke liye
extraReducers = async actions handle karne ke liye
🧩 Ek line mein

👉 return data → action.payload
👉 throw / rejectWithValue → action.error / payload
