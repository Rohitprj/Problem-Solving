TEST CASES (UNIT TEST CASES AND SYSTEM TEST CASES)

1. INTRODUCTION

Testing is one of the most important phases of software development. It ensures that the developed system works according to user requirements and performs correctly under different conditions. A test case is a set of conditions, inputs, steps, and expected results used to verify whether a software feature functions as intended. Formal test cases define inputs, execution steps, and expected outputs so tests can be repeated consistently.

For the Spiritual AI Chat Bot project, testing is necessary because the system includes user authentication, chatbot responses, voice queries, chat history, admin dashboard, and analytics. If these modules are not tested properly, users may face login failures, data loss, incorrect responses, or security issues.

Testing in this project is divided into:

1.1 Unit Testing – testing individual modules or functions separately  
1.2 System Testing – testing the complete integrated application as a whole  

Unit testing validates isolated components or source code behavior.

2. OBJECTIVES OF TESTING

The objectives of testing in this project are:

2.1 Verify correctness of all modules  
2.2 Detect bugs and errors early  
2.3 Ensure smooth user experience

3. UNIT TEST CASES

3.1 Meaning of Unit Testing

Unit testing checks the smallest testable parts of software such as functions, methods, or modules independently. It is generally performed by developers during coding.

For this project, unit testing is performed on:

3.1.1 Authentication functions  
3.1.2 Input validation functions  
3.1.3 Chat save logic  
3.1.4 Token generation  
3.1.5 Role verification  
3.1.6 Voice conversion handlers  
3.1.7 Analytics counters  

3.2 Unit Test Case Format

Test Case ID | Module | Test Description | Input | Expected Result | Status

4. SYSTEM TEST CASES

4.1 Meaning of System Testing

System testing validates the complete integrated system from end to end to ensure all modules work together correctly. It checks the whole application rather than individual units.

For this project, system testing includes:

4.1.1 User registration to chatbot usage  
4.1.2 Login to dashboard access  
4.1.3 Chat flow with database save  
4.1.4 Voice query to response flow  
4.1.5 Admin monitoring system  
4.1.6 Logout and session expiry  

4.2 System Test Case Format

Test Case ID | Scenario | Steps | Expected Result | Status  

4.3 System Test Cases for Spiritual AI Chat Bot
