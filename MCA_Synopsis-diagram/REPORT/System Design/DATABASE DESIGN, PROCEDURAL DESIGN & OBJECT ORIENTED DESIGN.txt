DATABASE DESIGN, PROCEDURAL DESIGN / OBJECT ORIENTED DESIGN

1. INTRODUCTION

System design is one of the most important phases of software development. It transforms user requirements into a technical blueprint that developers can implement efficiently. A good system design defines how data will be stored, how program modules will behave, and how components will interact. In the Spiritual AI Chat Bot project, system design is divided into three major areas:

1.1 Database Design – how data is organized and stored  
1.2 Procedural Design – step-by-step logic of modules and functions  
1.3 Object Oriented Design – classes, objects, and their interactions  

These three approaches together create a robust, maintainable, scalable, and secure application architecture.

2. DATABASE DESIGN

2.1 Meaning of Database Design

Database design is the process of organizing data according to a data model so that required information can be stored, retrieved, and managed efficiently. It determines what data is stored and how data elements relate to each other.

For the Spiritual AI Chat Bot project, database design is essential because the system stores:

2.1.1 User accounts  
2.1.2 Password hashes  
2.1.3 Chat conversations  
2.1.4 Voice query logs  
2.1.5 Admin records  
2.1.6 Reports and analytics  
2.1.7 Spiritual knowledge resources  

A well-designed database ensures fast performance, data integrity, and future scalability.

2.2 Database Used

The project uses MongoDB, a NoSQL document database.

2.3 Why MongoDB Was Chosen

2.3.1 Flexible schema structure  
2.3.2 Fast JSON-like document storage  
2.3.3 Easy integration with Node.js  
2.3.4 Good performance for chat systems  
2.3.5 Scalable for growing users  
2.3.6 Suitable for modern web/mobile apps

3. PROCEDURAL DESIGN

3.1 Meaning of Procedural Design

Procedural design converts system structure into step-by-step procedures, functions, and logic. It describes how tasks are executed through sequences, conditions, and loops. Procedural design traditionally follows function-oriented decomposition.

In this project, procedural design is used for backend logic, APIs, validations, and workflows.

3.2 Major Procedures in the Project

3.2.1 User Registration Procedure

Steps:
3.2.1.1 Accept name, email, password  
3.2.1.2 Validate input  
3.2.1.3 Check duplicate email  
3.2.1.4 Hash password  
3.2.1.5 Save user record  
3.2.1.6 Return success message  

3.2.2 User Login Procedure

Steps:
3.2.2.1 Accept email and password  
3.2.2.2 Search user in database  
3.2.2.3 Compare hashed password  
3.2.2.4 Generate JWT token  
3.2.2.5 Return login response  

3.2.3 Chat Processing Procedure

Steps:
3.2.3.1 Accept user message  
3.2.3.2 Validate text  
3.2.3.3 Send to AI engine  
3.2.3.4 Receive response  
3.2.3.5 Save conversation  
3.2.3.6 Return reply to user  

3.2.4 Voice Query Procedure

Steps:
3.2.4.1 Record speech  
3.2.4.2 Convert speech to text  
3.2.4.3 Send text to chatbot  
3.2.4.4 Generate answer  
3.2.4.5 Show result  

3.2.5 Admin Report Procedure

Steps:
3.2.5.1 Verify admin token  
3.2.5.2 Fetch analytics data  
3.2.5.3 Count users/chats  
3.2.5.4 Display dashboard reports  

3.3 Procedural Modules Used

3.3.1 authController()

3.3.3 registerUser()

3.4 Advantages of Procedural Design in This Project

3.4.1 Easy to understand workflows  
3.4.2 Good for API endpoints

3.4.4 Clear step-by-step debugging  
3.4.5 Useful for validation systems  

4. OBJECT ORIENTED DESIGN (OOD)

4.1 Meaning of Object Oriented Design

Object-oriented design solves software problems by building systems as interacting objects and classes. It uses principles such as encapsulation, inheritance, and polymorphism to model real-world entities.

This approach is highly useful for scalable applications.

4.2 Why OOD is Suitable for This Project

The Spiritual AI Chat Bot contains many real-world entities:

4.2.1 User  
4.2.2 Admin  
4.2.3 Chat Session  
4.2.4 Message  
4.2.5 AI Assistant  
4.2.6 Report  
4.2.7 Notification  

These can be represented naturally as classes and objects.

5. COMPARING PROCEDURAL AND OBJECT ORIENTED DESIGN

Basis: Procedural Design vs Object Oriented Design

5.1 Focus – Functions vs Objects  
5.2 Best For – APIs/workflows vs Large scalable systems  
5.3 Reuse – Moderate vs High  
5.4 Maintenance – Medium vs High  
5.5 Real-world modeling – Low vs Strong  

6. COMBINED APPROACH USED IN THIS PROJECT

In This Project:

This hybrid approach is practical and professional.
