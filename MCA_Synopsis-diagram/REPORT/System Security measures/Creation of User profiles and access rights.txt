CREATION OF USER PROFILES AND ACCESS RIGHTS

1. INTRODUCTION

In modern software systems, not all users should have the same permissions. Different users require different levels of access based on their roles and responsibilities.

1.1 The process of creating user accounts with defined permissions is called Creation of User Profiles and Access Rights  
1.2 It is a core security mechanism used to protect system data and resources  
1.3 It ensures only authorized actions are allowed within the application  

1.4 For the Spiritual AI Chat Bot project:

1.4.1 Users interact with chatbot features  
1.4.2 Admins manage system data and monitor activity  
1.4.3 Separate access levels are required for security  

1.5 Role-Based Access Control (RBAC) is used:

1.5.1 Permissions are assigned to roles, not individuals  
1.5.2 It follows the principle of least privilege

2.7 Support system scalability

3. MEANING OF USER PROFILE

3.1 A User Profile is a stored account record that identifies a system user  

3.2 Typical profile data includes:

3.2.1 Full Name  
3.2.2 Email Address  
3.2.3 Password (hashed)  
3.2.4 Role Type  
3.2.5 Profile Image (optional)  
3.2.6 Account Status  
3.2.7 Created Date  
3.2.8 Last Login Time  

3.3 Each registered user gets a unique profile after signup  

---

4. MEANING OF ACCESS RIGHTS

4.1 Access Rights define what a user is allowed to do in the system  

4.2 Examples of permissions:

4.2.1 Read data  
4.2.2 Create data  
4.2.3 Update profile  
4.2.4 Delete records  
4.2.5 View reports  
4.2.6 Manage users  
4.2.7 Access admin dashboard  

4.3 RBAC structure:

4.3.1 User → Role → Permissions
