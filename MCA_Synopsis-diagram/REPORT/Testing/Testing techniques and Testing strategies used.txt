TESTING TECHNIQUES AND TESTING STRATEGIES USED

1. INTRODUCTION

Testing is one of the most important phases of software development. It is the process of evaluating a software system to identify defects, verify correctness, and ensure that the final product meets user requirements.

1.1 The objective of testing is not only to find bugs but also to improve quality, reliability, performance, and security.  

1.2 For the Spiritual AI Chat Bot project, testing is essential because the system includes multiple modules such as:  
1.2.1 User registration and login  
1.2.2 JWT authentication  
1.2.3 AI chatbot responses  
1.2.4 Chat history management  
1.2.5 Voice query support  
1.2.6 Admin dashboard  
1.2.7 Reports and analytics  
1.2.8 Mobile user interface  

1.3 To ensure the system works correctly, various testing techniques and testing strategies are used.

1.4 Testing strategies commonly include levels such as unit, integration, and system testing, while techniques include black-box and white-box methods.

---

2. MEANING OF TESTING TECHNIQUES

2.1 Testing techniques are methods used to design and execute test cases in order to validate software behavior.

2.2 They help testers decide:
2.2.1 What to test  
2.2.2 How to test  
2.2.3 Which inputs to use  
2.2.4 How to detect defects efficiently  

2.3 Software testing techniques commonly include:
2.3.1 Manual testing  
2.3.2 Automated testing  
2.3.3 Functional testing  
2.3.4 Non-functional testing  

---

3. MEANING OF TESTING STRATEGIES

3.1 Testing strategies are planned approaches used to organize and manage the overall testing process.

3.2 They define:
3.2.1 Testing levels  
3.2.2 Order of testing  
3.2.3 Tools used  
3.2.4 Resources required  
3.2.5 Scope of testing  
3.2.6 Risk priorities  

3.3 A testing strategy ensures testing is systematic rather than random.

PART A – TESTING TECHNIQUES USED

5. BLACK BOX TESTING

5.1 Black box testing checks system functionality without looking at internal code.

5.2 It focuses on inputs and outputs only.

5.3 Used in this project:
5.3.1 Login testing  
5.3.2 Signup testing  
5.3.3 Chat response validation  
5.3.4 Voice query testing  
5.3.5 Admin dashboard functions  

5.4 Example:
5.4.1 Input: Email + Password  
5.4.2 Output: Login Success / Failure  

---

6. WHITE BOX TESTING

6.1 White box testing checks internal code logic, conditions, loops, and paths.

6.2 Used in this project:
6.2.1 Authentication logic  
6.2.2 Token validation functions  
6.2.3 Role checking middleware  
6.2.4 Database service functions  
6.2.5 API error handling logic  

6.3 Example:
6.3.1 Testing password hashing correctness  

---

7. UNIT TESTING

7.1 Unit testing checks individual functions or components in isolation.

7.2 Used in this project:
7.2.1 validateEmail()  
7.2.2 generateJWT()  
7.2.3 saveChat()  
7.2.4 comparePassword()  
7.2.5 fetchHistory()  

7.3 Example:
7.3.1 validateEmail("abc@gmail.com") = true  

---

8. INTEGRATION TESTING

8.1 Integration testing verifies that modules work correctly together.

8.2 Used in this project:
8.2.1 Login API + Database  
8.2.2 Chat API + AI Engine  
8.2.3 Voice input + Speech-to-text + Chatbot  
8.2.4 Dashboard + Analytics APIs  

8.3 Example:
8.3.1 User Login → Token Generated → Dashboard Opened
