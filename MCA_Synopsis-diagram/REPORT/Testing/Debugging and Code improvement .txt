2. MEANING OF DEBUGGING

2.1 Debugging is a systematic process used to identify and fix errors in software code.

2.2 The debugging process includes:
2.2.1 Reproducing the issue  
2.2.2 Locating faulty code  
2.2.3 Identifying root cause  
2.2.4 Fixing the issue  
2.2.5 Re-testing the solution  
2.2.6 Documenting the fix  

2.3 This stepwise approach ensures reliability and correctness of the system.

6. DEBUGGING EXAMPLES FROM THE PROJECT

6.1 LOGIN FAILURE BUG

6.1.1 Problem:
Correct users could not log in  

6.1.2 Cause:
Password comparison used plain text instead of bcrypt  

6.1.3 Fix:
Implemented proper bcrypt password comparison  

---

6.2 CHAT HISTORY NOT SHOWING

6.2.1 Problem:
Old chats were not loading  

6.2.2 Cause:
Incorrect userId passed in API  

6.2.3 Fix:
JWT decoded properly and correct userId used  

---

6.3 ADMIN ROUTE ACCESS ISSUE

6.3.1 Problem:
Normal users accessed admin panel  

6.3.2 Cause:
Role middleware missing  

6.3.3 Fix:
Added role-based authorization middleware  

---

6.4 SLOW ANALYTICS REPORTS

6.4.1 Problem:
Dashboard reports loaded slowly  

6.4.2 Cause:
Unoptimized database query  

6.4.3 Fix:
Added indexing and query optimization
